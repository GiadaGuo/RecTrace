Flink 里跨域序列是怎么合并的？
agent
→ get_pipeline_status(job="sequence_merge")
Job B 用 Flink CoProcessFunction 做双流 join：主行为流（behavior-events）和广告曝光流（ad-expose-events）按 user_id 做 keyBy，在 5 分钟 EventTime 窗口内合并。合并后的跨域序列写入 Redis hash，key 格式为 user:{id}:cross_seq，TTL 30min。
Data flow
Mock Generator → Kafka
Kafka → Flink Job A (stat features)
Kafka → Flink Job B (seq + cross-domain)
Flink → Redis (KV, TTL 30min)
Request → FastAPI → Redis → LLM rank → SSE
Key design decisions
LLM 做排序而非召回，避免实时推理延迟过高
序列截断 len=50，控制 prompt token 数
跨域用 CoProcessFunction 双流 join
Agent 工具函数直查 Redis，无额外延迟
SQLite 仅存物品元数据，不参与实时链路
